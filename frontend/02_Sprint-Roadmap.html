기술 스택 행안전부 공중화장실 API TourAPI (행사/팝업) Firebase Cloud Messaging Supabase (화장실 DB)
작업 항목 구현 상세 / 노트 난이도

Sprint 3 — 혼잡도 + 루트 최적화 (Vibe-Route & Crowd Level)
Feature 3 완성 · 혼잡도 신호등 · 루트 생성 · 이동 시간 계산

목표 장소별 혼잡도 신호 표시 + 사용자가 직접 루트를 구성하고 예상 이동 시간을 확인할 수 있는 기능 완성.
기술 스택 TourAPI (방문자 통계) Kakao Mobility API Supabase (routes, favorites) React DnD
작업 항목 구현 상세 / 노트 난이도

TourAPI stayList (숙박통계), 주요 POI 월별 방문자 수 적재. GitHub Actions Cron (매주 일요일) 자동 수집. crowd_stats 테이블 설계 (place_id, dayofweek, hour, level).

현재 요일+시간대 → DB 조회 → 하위 33% = 여유(녹), 중간 = 보통(황), 상위 33% = 혼잡(적). 실시간 데이터 없을 경우 평균값 fallback. 1시간 캐시.

 쉬움
장소 즐겨찾기 → 루트 생성 (순서 드래그)
즐겨찾기 장소 목록 → "루트 만들기" 진입. react-beautiful-dnd로 순서 드래그. 최대 10개 장소. 루트 Supabase 저장 (routes 테이블).
 보통
TourAPI 인증 숙소 / 맛집 연결
루트 내 장소에 근접한 TourAPI 인증 숙소(contentTypeId=32) / 맛집(39) 추천 카드 표시. 루트 종료 지점 기준 반경 1km 이내.
 쉬움
예상 이동 시간 계산
1안: Kakao Mobility API future/route (유료, 정확). 2안: Haversine 직선거리 + 도보 4km/h 환산 (무료). MVP는 2안 → 공모전 후 1안으로 업그레이드. 루트 카드에 총 예상 시간 표시.
 보통
완료 기준 (DoD)

Sprint 4 — SNS Spot Analyzer (Trend Spot & Pop-up)
Feature 1 완성 · YouTube URL → AI 장소 분석 → 지도 핀 연결
Week 8–9 (2주) 주로 보통 ★ 핵심 차별화
목표 YouTube URL을 입력하면 영상 제목·설명에서 AI가 장소명을 추출하고 지도에 "SNS 트렌드" 핀으로 표시. 10초 내 결과 제공.
기술 스택 YouTube Data API v3 FastAPI (Python) OpenAI gpt-4o-mini TourAPI /searchKeyword Render.com Upstash Redis (URL 해시 캐싱)
작업 항목 구현 상세 / 노트 난이도
YouTube URL 파싱 → videos.list API 메타데이터 추출
URL에서 videoId 추출 (정규식). YouTube Data API videos.list로 snippet(제목, 설명, 태그) 취득. API 키 1개당 일 10,000 quota.
 보통
FastAPI AI Worker 구축 (Render.com 배포)
Next.js → FastAPI 별도 서비스 분리 (AI 처리 격리). /analyze POST 엔드포인트. Render.com Free Tier 배포 (슬립 문제 → Sprint 7에서 Cron으로 해결). Docker 없이 Python 직접 배포.
 보통
OpenAI gpt-4o-mini: 영상 제목/설명에서 장소명 추출
프롬프트 설계: "다음 텍스트에서 한국 실제 장소명을 JSON 배열로 추출하세요." JSON Schema 출력 강제. 모델: gpt-4o-mini (비용 최소화, ~$0.001/건). 최대 10개 장소 추출.
 보통
추출된 장소명 → TourAPI /searchKeyword 매핑
장소명 배열 → TourAPI searchKeyword 병렬 호출 (Promise.allSettled). 매핑 실패 시 제외. 결과: [{name, lat, lng, contentId}] 배열 반환.
 보통
분석 결과 캐싱 (URL 해시 기반 중복 방지)
YouTube videoId → SHA-256 해시 → Redis 키. TTL 24시간. 동일 URL 재분석 시 캐시 히트 → OpenAI 호출 0건. 캐시 히트율 목표 70%+.
 보통
분석 결과 지도 핀 표시 + "SNS 트렌드" 뱃지
일반 POI 핀과 구분되는 보라색 "트렌드" 마커. 핀 클릭 → 원본 YouTube 영상 링크 + 장소 상세. 분석 결과 카드 (영상 썸네일, 제목 포함).
 쉬움
quota 소진 모니터링 + 슬랙/이메일 알림
YouTube API / OpenAI API 일 사용량 추적 → 80% 도달 시 Slack Webhook 알림. 긴급 시 분석 기능 자동 비활성화 플래그 (Feature Flag).
 보통
완료 기준 (DoD)
YouTube URL 입력 → 10초 내 장소 핀 지도 표시. 동일 URL 재입력 시 즉시(캐시) 결과.
리스크
gpt-4o-mini 장소 추출 정확도 ~70% → 프롬프트 Few-shot 예시 추가로 개선 필요
의존성
Sprint 0 (FastAPI 환경변수), Sprint 1 (TourAPI 캐싱)

목표 스타·드라마·무드 태그를 선택하면 AI가 관련 장소를 추천하고 "○○의 하루 루트" 카드를 생성. 루트 저장 및 SNS 공유 지원.
기술 스택 OpenAI gpt-4o-mini TourAPI /searchKeyword Supabase (persona_maps, routes) Vercel OG Image (공유 카드)
작업 항목 구현 상세 / 노트 난이도
페르소나 선택 UI (스타/드라마/무드 태그)
3단계 선택 UI: ① 카테고리 (K-POP 아이돌 / K-드라마 / 무드) → ② 세부 선택 (예: BTS / 이상한 변호사 우영우 / 로맨틱) → ③ 선택 확인. 카드 그리드 UI, 이미지 포함.
 쉬움

수동 큐레이션 50개 (예: BTS → 한강공원, 광화문광장 / 이상한변호사우영우 → 통영 등). Supabase persona_places 테이블. 관리자 페이지에서 추가 가능.
 보통
OpenAI: 무드 태그 → 장소 카테고리 매핑 프롬프트
입력: 무드 태그 배열 + 현재 위치 도시. 출력: 추천 장소 카테고리 + 키워드 배열 (JSON). 예: "로맨틱 + 서울" → ["한강 야경", "카페거리", "전망대"]. 품질 보장을 위한 프롬프트 반복 테스트 필요.
 어려움
매핑된 카테고리로 TourAPI 장소 필터링 + 루트 생성
